## 📖 Overview
This project demonstrates the process of capturing and analyzing network traffic using **Wireshark**, the industry-standard network protocol analyzer. The primary objective was to observe unencrypted HTTP communication, deeply inspect captured packets, and understand the mechanics of data exchange between a client and a remote web server.

The packet capture was executed within a Kali Linux environment. HTTP packets were systematically filtered and analyzed to study standard request/response messages and status codes.

## 🎯 Objective
* Install and configure Wireshark for packet sniffing.
* Capture live network traffic on an active interface.
* Utilize display filters to isolate specific protocols (HTTP).
* Analyze packet headers, protocol information, and IP routing.
* Document and export the captured traffic for future forensic analysis.

## 🛠️ Tools Used
* **Operating System:** Kali Linux
* **Network Analyzer:** Wireshark
* **Protocols Observed:** HTTP, TCP, IPv4
* **Capture Format:** `.pcapng`

---

## 🚀 Capture Procedure
1. Launched Wireshark on Kali Linux.
2. Selected the active network interface (`eth0`) and initiated the packet capture in promiscuous mode.
3. Generated unencrypted HTTP traffic using a standard web browser.
4. Applied the `http` display filter within Wireshark to cut through background noise and isolate relevant packets.
5. Inspected the details of the HTTP request and response frames.
6. Exported the session as a PCAPNG file for documentation.

---

## 🔍 Packet Analysis

### HTTP GET Request
The captured traffic reveals an initial HTTP GET request generated by the browser seeking to retrieve a web resource.

| Field | Value |
| :--- | :--- |
| **Protocol** | HTTP |
| **Request Method** | `GET` |
| **HTTP Version** | HTTP/1.1 |
| **Source IP** | `172.20.10.14` |
| **Destination IP** | `34.223.124.45` |
| **Destination Port** | `80` |
| **Host** | `detectportal.firefox.com` |

> *Note: This specific request demonstrates the browser performing an automated check to determine if the network is restricted by a captive portal.*

### HTTP Server Responses
The capture contains multiple server responses, highlighting standard HTTP communication flow:

* **`HTTP/1.1 200 OK`**: Indicates that the requested resource was successfully found and delivered to the client.
* **`HTTP/1.1 301 Moved Permanently`**: Indicates that the requested resource has been permanently redirected to a new location, instructing the browser to update its target URL.

### Additional HTTP Requests
As the webpage loaded, the capture recorded automated follow-up requests utilized to fetch supplementary resources:
* `GET /`
* `GET /online`
* `GET /favicon.ico`
