## Overview

This document provides a detailed analysis of the HTTP packets captured using Wireshark during Task 8 of the OIBSIP Cybersecurity Internship. The purpose of this analysis is to understand how HTTP communication occurs between a client and a web server.

# Observed Packets

The packet capture contains several HTTP requests and responses exchanged between the client and remote web servers.

### HTTP GET Request

The browser initiated an HTTP GET request to request a webpage from the server.

**Observed Details**

* HTTP Method: `GET`
* HTTP Version: `HTTP/1.1`
* Destination Port: `80`
* Host: `detectportal.firefox.com`
* Client IP: `172.20.10.14`
* Server IP: `34.223.124.45`

This packet represents the beginning of the HTTP communication process.

---

### HTTP Response – 200 OK

The server responded with the status code:

```text
HTTP/1.1 200 OK
```

This indicates that:

* The request was successfully processed.
* The requested resource was found.
* The server returned the requested content to the client.

---

### HTTP Response – 301 Moved Permanently

Another response captured was:

```text
HTTP/1.1 301 Moved Permanently
```

This response indicates that:

* The requested resource has been permanently moved.
* The client should use the new URL provided by the server.
* Browsers automatically follow this redirect.

---

### Additional Requests

Additional requests observed include:

* `GET /`
* `GET /online`
* `GET /favicon.ico`

These are common browser requests generated while loading webpage resources.

# Protocol Analysis

## TCP

TCP provided reliable communication between the client and server by ensuring packet delivery, sequencing, and acknowledgments.

## HTTP

HTTP is an application-layer protocol used for exchanging web content. Since HTTP traffic is transmitted without encryption, request headers and response information are visible in Wireshark.
